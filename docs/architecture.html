直播录制架构 · 2 轴终态

平台专属的一切归平台(取流 + 弹幕);下载是平台无关的共享引擎。契约在 core,其余全通用。

① 平台轴 — 每平台一个包

<平台>-live:取流(URL→流地址 + headers)、判活、主播名、弹幕连接。一切平台专属。接新平台 = 只写这一个包。

② 引擎轴 — 共享横切(只一份)

record-engine:通用 PollingRecorder 编排 + 下载引擎实现(ffmpeg / mesio …)。平台无关,所有平台共用。Engine 接口 + 注册表在 core。任务字段 engine 选。

接新平台(如虎牙)= 只写一个 huya-live 包(实现 getStream + 可选 connectDanmu + matchUrl)。引擎(ffmpeg/mesio)白嫖 record-engine,无需写任何录制器;弹幕协议不同则在该包内实现。
加新引擎(如 yt-dlp)= 只在 record-engine 里加一个实现 Engine 接口的策略,所有平台立即可用。
